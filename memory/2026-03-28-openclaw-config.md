## Нормальные варианты

### 1) **OpenClaw `memorySearch.provider = "local"`**
Это встроенный локальный путь OpenClaw.

**Как работает:**
- OpenClaw сам использует локальную embedding-модель
- индекс хранит у себя
- без API ключей

**Плюсы:**
- нативно для OpenClaw
- без внешних сервисов
- приватно

**Минусы:**
- может потребовать нативную сборку / model download
- иногда капризнее в первичной настройке

**Когда брать:**
- хочешь “по канону OpenClaw”
- ок один раз нормально настроить

---

### 2) **Ollama + embeddings**
OpenClaw умеет `memorySearch.provider = "ollama"`.

**Как работает:**
- Ollama крутится локально
- OpenClaw шлёт ему embedding-запросы
- всё остаётся на машине

**Плюсы:**
- очень понятная схема
- проще дебажить
- можно менять модели без ковыряния OpenClaw-внутренностей

**Минусы:**
- нужен установленный и запущенный Ollama
- ещё один сервис в системе

**Когда брать:**
- если Ollama уже стоит
- если хочешь самый понятный локальный стек

---

### 3) **QMD backend**
Это уже посерьёзнее: локальный sidecar для memory/search.

**Плюсы:**
- мощнее по поиску
- локально
- может быть лучше на больших объёмах заметок

**Минусы:**
- самый сложный вариант
- для “просто включить память” — избыточно

**Когда брать:**
- только если хочешь именно прокачанный memory/search стек
- не как первый шаг
