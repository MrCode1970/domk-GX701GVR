# fox — Идеи автоматизации

Где можно ботить, где надо сохранить ручность. Конкретные сценарии.

---

## 🎯 Принцип: Automation ≠ All

**Ботим:**
- Повторяющиеся механические операции
- Когда правило ясно и нет исключений

**Не ботим:**
- Качество фильтрации (нужно суждение)
- Творчество (формулирование решений)
- Приоритизацию (выбор что важно)

## 🔄 СЕЙЧАС ГОТОВО (Low-hanging fruit)

### 1️⃣ Git Auto-Commit (простая автоматизация)

**Проблема:** Забываю коммитить, история теряется между сессиями  
**Решение:** File watcher → автоматический коммит при изменении kb/

**Как работает:**
```bash
# При изменении любого файла в kb/
watch kb/ → git add kb/ → git commit -m "auto: Update KB"
```

**Инструмент:** VSC задача + git hook (pre-commit или post-merge)

**Установка (простая):**
```bash
# Создать .git/hooks/post-commit
#!/bin/bash
if git diff --name-only HEAD~1..HEAD | grep -q "kb/"; then
  echo "[Auto] KB изменён, можно синхронизировать"
fi
```

**Результат:** История всегда синхронизирована ✅

---

### 2️⃣ Daily Memory Archive (расписание)

**Проблема:** Дневник `memory/YYYY-MM-DD.md` растёт, нужно архивировать старое  
**Решение:** Cron job → архивировать старые дневники

**Как работает:**
```bash
# Каждый день в 23:59
cron: 23 59 * * * /scripts/archive-daily-memory.sh

# Скрипт:
# 1. Посмотреть файлы в memory/ старше 7 дней
# 2. Переместить в memory/archive/2026-03/
# 3. Создать новый пустой дневник для сегодня
```

**Результат:** memory/ всегда аккуратная, архив организован ✅

---

### 3️⃣ Weekly KB Review Reminder (уведомление)

**Проблема:** Забываю еженедельный review (проверить что архивировать, что добавить)  
**Решение:** Heartbeat или cron → напоминание в Discord

**Как работает:**
```bash
# Каждый понедельник в 09:00
cron: 0 9 * * 1 /scripts/weekly-review.sh

# Скрипт:
# 1. Подсчитать новые задачи за неделю
# 2. Вывести их список
# 3. Отправить сообщение в Discord: "Review: N задач решено"
```

**Где отправлять:** #fox-chat или вам в DM  
**Результат:** Еженедельный срез, не пропускаем ничего ✅

## 🔜 ПОТОМ МОЖНО БОТИТЬ (Medium effort)

### 4️⃣ Discord → KB Exporter (semi-auto)

**Проблема:** Важные выводы из Discord тредов нужно выносить в KB вручную  
**Решение:** Bot команда или реакция → автоматическая выгрузка

**Вариант А: Bot команда**
```
User: !export [thread_id] [filename] [tags]

Bot:
1. Читает тред
2. Выгружает в kb/projects/fox/[filename].md
3. Добавляет metadata (дата, теги, Discord ссылка)
4. Коммитит в git
5. Отвечает: "✅ Выгружено в KB"
```

**Вариант Б: Реакция на сообщение**
```
User: Реагирует 📋 на сообщение в Discord

Bot:
1. Видит реакцию
2. Выгружает сообщение + тред в KB
3. Сохраняет в архив
4. Создаёт коммит
```

**Сложность:** Нужны разрешения бота (есть), нужна логика классификации  
**Выигрыш:** Экономия 30 минут в неделю на выгрузке

**Результат:** KB наполняется по требованию ⚡

---

### 5️⃣ MEMORY.md Auto-Update (ежедневно)

**Проблема:** MEMORY.md может отставать от текущего контекста  
**Решение:** Daily script → обновить ключевые факты

**Как работает:**
```bash
# Каждый день в 23:30
cron: 30 23 * * * /scripts/update-memory.sh

# Скрипт:
# 1. Прочитать memory/[сегодня].md
# 2. Извлечь ключевые события (status: [emoji] [text])
# 3. Обновить MEMORY.md с новыми фактами
# 4. Сохранить старые факты в backlog
```

**Сложность:** Нужна парсинг daily log'ов (regex + heuristics)  
**Выигрыш:** MEMORY всегда актуальная ✅

---

### 6️⃣ Git History Analytics (еженедельно)

**Проблема:** Хочется видеть как быстро мы работаем, какие файлы меняются чаще  
**Решение:** Script → анализ git log + отчёт

**Как работает:**
```bash
# Каждый понедельник в 10:00
cron: 0 10 * * 1 /scripts/git-analytics.sh

# Скрипт:
# 1. git log --stat за последние 7 дней
# 2. Подсчитать коммиты по категориям
# 3. Вывести: "Решено 5 задач, 23 коммита, 450 строк текста"
# 4. Сохранить в kb/projects/fox/weekly-stats.md
```

**Результат:** Видим velocity и активность 📈

## 🤖 ПОЗЖЕ МОЖНО БОТИТЬ (High effort)

### 7️⃣ Discord Conversation Threading Auto-Archive (complex)

**Идея:** Когда тред помечен как "решено" → автоматически выносит в KB

**Сложность:** Нужно определить что тред "завершён" (можно по emoji или команде)

**Выигрыш:** Полная автоматизация KB наполнения

---

### 8️⃣ AI-powered Summarization (очень complex)

**Идея:** Длинный тред → AI генерирует краткий summary для KB

**Сложность:** 
- Нужен LLM access (OpenRouter уже есть)
- Нужны правила что summarize, что нет
- Может "потерять" важные детали

**Выигрыш:** Экономия времени на написании выводов  
**Риск:** Неправильный summary в KB

## 📋 ОСТАЁТСЯ РУЧНЫМ (And should be!)

### ❌ Не ботим: Фильтрация качества

**Почему:** То что важно — зависит от контекста, требует суждения  
**Пример:** Из 10 идей в треде 3 стоят внимания → нужно выбрать какие

### ❌ Не ботим: Редактирование решений

**Почему:** KB должна быть красиво написана, не сырой выгруз  
**Пример:** Сырой export из Discord → нужна доделка и форматирование

### ❌ Не ботим: Приоритизация задач

**Почему:** Только вы знаете что важнее  
**Пример:** Что сначала: VSC интеграция или Discord ботинг?
