# fox — Порядок работы (Workflow)

Как мы работаем вместе. Логика процесса для вас и для меня.

---

## 🎯 Принцип работы

**Вы** — принимаете решения, задаёте направление, видите результаты  
**Я** — анализирую, документирую, помню контекст, предлагаю решения  
**Discord** — место обсуждения (эфемерное)  
**KB** — место истины (вечное, версионируется)  

---

## 📋 Цикл работы (как решается одна задача)

### 1️⃣ ПОСТАНОВКА (вы → я)

**Где:** Discord #fox-chat или другой тред  
**Что происходит:**
```
Вы: "Нужно сделать X"
 ↓
Я: Уточняю (1-2 вопроса если неясно)
 ↓
Вы: Подтверждаете или правите
```

**Я документирую:**
- Дату, время в `memory/YYYY-MM-DD.md`
- Суть задачи: что, зачем, когда нужно
- Статус: `📋 Queued`

### 2️⃣ РЕШЕНИЕ (я работаю)

**Где:** VSC, локально, или Discord тред  
**Что происходит:**
```
Я читаю:
  - Контекст (MEMORY.md)
  - Похожие задачи (kb/projects/fox/tasks.md)
  - Текущий статус (WORKLOG.md)
 ↓
Я разбираюсь:
  - Собираю информацию
  - Пробую решения
  - Сужу логику
 ↓
Я предлагаю:
  - Вариант 1 / вариант 2 / вариант 3
  - Или готовое решение если очевидно
```

**Я документирую (live):**
- Шаги в `memory/YYYY-MM-DD.md`
- Попытки и ошибки (для learnings)
- Статус: `🔄 In Progress`

### 3️⃣ РЕЗУЛЬТАТ (я → вы)

**Где:** Discord или VSC (если большой)  
**Что происходит:**
```
Я: Вот решение: XXX (вариант А / вариант Б / готово)
 ↓
Вы: OK / правки / отклоняю
 ↓
(если OK) → фаза 4
(если правки) → цикл повторяется, я уточняю
```

**Я документирую:**
- Решение: формулирую чётко
- Статус: `✅ Solved` или `🔧 Needs refinement`

### 4️⃣ АРХИВ (я)

**Где:** KB  
**Что происходит:**
```
Я выношу в KB:
 ├─ kb/projects/fox/tasks.md (добавляю в "✅ Решённые")
 ├─ kb/projects/fox/learnings.md (если был урок)
 ├─ kb/projects/fox/decisions.md (если было архрешение)
 └─ [или создаю специальный файл если нужно]

Структура записи:
  - Название задачи
  - Что было (проблема)
  - Как решили (решение)
  - Почему (логика)
  - Дата и ссылка на Discord тред
  - Теги

Коммитю в git:
  git add kb/
  git commit -m "Add: [название задачи] (дата)"
  
Результат:
  - KB обновлена (истина)
  - Git история ведётся (версионирование)
  - MEMORY.md обновляю (ключевые факты)
```

**Я документирую:**
- Статус: `✅ Archived`
- Ссылка: на файл в KB

## 🗂️ Режимы памяти (когда что)

| Память | Зачем | Когда | Кто | Как долго |
|--------|-------|-------|-----|----------|
| **memory/YYYY-MM-DD.md** | Журнал событий дня | Every day | Я | 7 дней, потом архив |
| **MEMORY.md** | Контекст для восстановления | При загрузке сессии | Я | Постоянно (обновляется) |
| **kb/** | Истина, решения, опыт | После решения задачи | Я | Вечно (git history) |
| **.vscode/** | Мой рабочий инструмент | При работе | Я | Постоянно |

## 🎬 Пример работы (реальный)

### Задача: "Организовать структуру Discord"

**Фаза 1: Постановка (Discord)**
```
mr1970: "Наведи порядок на сервере"
Лис: "Удалить мусор, переорганизовать каналы?"
mr1970: "Да, удаляй"
```

**Я записал:** `memory/2026-03-31.md`
```
- Задача: организовать Discord структуру
- Статус: 📋 Queued → 🔄 In Progress
- Дата: 2026-03-31 04:40
```

**Фаза 2: Решение (я работаю)**
```
1. Смотрю текущую структуру (channel-list)
2. Удаляю мусор (3 канала)
3. Создаю категории (3 шт)
4. Перемещаю каналы
5. Обновляю описания (topic)
```

**Я записал:** `memory/2026-03-31.md`
```
- Действия:
  - Удалено 3 канала (fox-chat форум, bot-discussion, тестовый-болталка)
  - Создано 3 категории (ОСНОВНОЕ, DOCS, СЕРВИС)
  - Перемещено 8 каналов
  - Обновлены описания
- Результат: ✅ Готово
```

**Фаза 3: Результат (я → вы)**
```
Я: "Порядок наведён. Структура готова. (отправляю скрин/описание)"
Вы: "Ок, хорошо"
```

**Я записал:** `memory/2026-03-31.md`
```
- Статус: ✅ Solved
- Результат: Discord структурирован
```

**Фаза 4: Архив (я)**
```
Обновляю kb/projects/fox/tasks.md:

### ✅ Чистка Discord структуры
**Дата:** 2026-03-31  
**Статус:** ✅ Done

**Проблема:**
Два `fox-chat`, тесты в разных каналах, структура неясна.

**Решение:**
- Удалены 3 мусорных канала
- Созданы 3 категории (ОСНОВНОЕ, DOCS, СЕРВИС)
- Каналы сгруппированы по смыслу
- Обновлены описания

**Результат:**
Discord готов к работе, структура ясна всем.

Коммитю:
  git add kb/projects/fox/tasks.md
  git commit -m "Add: Discord структура (2026-03-31)"
  
Обновляю MEMORY.md:
  - last_proven: Discord структура готова
  - next: VSC интеграция
```

## 🔄 Правила сохранения и синхронизации

### Git (версионирование истины)
```bash
# Когда:
  - После решения задачи
  - Обновления KB
  - Важных выводов

# Как:
  git add kb/
  git commit -m "Add/Update: [название] (YYYY-MM-DD)"
  
# История:
  git log --oneline kb/
  → видим эволюцию решений
```

### MEMORY.md (оперативная память)
```markdown
## Current state
- focus: [текущая задача]
- last_proven: [последний подтверждённый факт]
- next: [что дальше]
- unresolved: [непонятные моменты]

## Stable environment
- [факты об окружении]

## Stable rules
- [договорённости, правила]
```

### memory/YYYY-MM-DD.md (дневник)
```markdown
# 2026-03-31

## События дня
- 04:40 — обсуждение Discord структуры
- 04:56 — реорганизация сервера
- ...

## Выводы дня
- Структура каналов важна для ясности
- Порядок ускоряет работу

## На завтра
- Продолжить VSC интеграцию
```

## 📞 Контакт / Эскалация

**Если что-то не ясно:**
```
Я: "Уточню ещё раз..."
Вы: Отвечаете или ставите thumbs-up (да)
```

**Если я застрял:**
```
Я: "Не знаю как дальше"
Вы: Даёте направление или даёшь доступ/разрешение
```

**Если срочно нужна смена приоритета:**
```
Вы: В Discord "СРОЧНО" или ⚡ реакция
Я: Останавливаю текущее, берусь за срочное
```

## 💡 Итого

**Одна задача = 4 фазы:**
1. Постановка (вы четко говорите)
2. Решение (я разбираюсь)
3. Результат (я показываю, вы подтверждаете)
4. Архив (я сохраняю в KB + git)

**Вы видите:** результаты, диаграммы, вариант A/B/C  
**Я помню:** контекст, историю, связи между решениями  
**Discord:** живая работа  
**KB + Git:** вечная память
